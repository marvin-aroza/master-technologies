Styling & Forms

Best practices for building beautiful, validated, and performant user interfaces.

Modern Styling: Tailwind & CSS Modules

Choosing the right styling strategy depends on your team's needs for speed vs. isolation.

  • Tailwind CSS: Utility-first. Results in very fast development and small production bundles. Best for modern startups.
  • CSS Modules: Standard CSS scoped to the component. Best for large teams who prefer raw CSS control and standard selectors.
  • Design Tokens: Using CSS variables to store your brand's colors, spacing, and typography ensures consistency across both strategies.
Tip: If you use Tailwind, use the `tailwind-merge` and `clsx` libraries to handle dynamic class merging safely.
Performant Forms: React Hook Form & Zod

Manual state management for forms (`useState` on every input) causes unnecessary re-renders. High-performance forms use a specialized library.

  • React Hook Form: Uses "uncontrolled" inputs under the hood. Only re-renders the specific input being changed.
  • Zod Validation: A schema declaration library. It defines the "shape" of your data and provides automatic validation and TypeScript typing.
import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import * as z from "zod";

const schema = z.object({
  email: z.string().email(),
  password: z.string().min(8)
});

function LoginForm() {
  const { register, handleSubmit, formState: { errors } } = useForm({
    resolver: zodResolver(schema)
  });

  return (
    <form onSubmit={handleSubmit(d => console.log(d))}>
      <input {...register("email")} />
      {errors.email && <span>{errors.email.message}</span>}
      
      <input type="password" {...register("password")} />
      <button>Login</button>
    </form>
  );
}
