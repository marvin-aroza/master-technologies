Reliable Testing: RTL & MSW

Modern React testing focuses on user behavior rather than internal state.

  • React Testing Library (RTL): Provides a way to test components by finding elements the same way a user would (labels, text content).
  • MSW (Mock Service Worker): Mocks the network at the API level. This allows your tests to perform real fetch calls that are intercepted and returned with mock data, ensuring your data-fetching logic is actually exercised.
  • Playwright: Used for End-to-End (E2E) testing. It launches a real browser to verify that the entire app works from the user's login to the final action.
// RTL Test Example
import { render, screen } from '@testing-library/react';
import userEvent from '@testing-library/user-event';

test('click button increments count', async () => {
  render(<Counter />);
  const button = screen.getByRole('button', { name: /increment/i });
  await userEvent.click(button);
  expect(screen.getByText(/count: 1/i)).toBeInTheDocument();
});
Modern Routing: React Router 7 & TanStack Router

Routing has evolved from simple path matching to managing full data lifecycles (loaders and actions).

  • React Router v7: The standard choice. It now integrates features from Remix, allowing you to fetch data *before* the component even renders via "Loaders."
  • TanStack Router: A newer, 100% typesafe router. It provides incredible intellisense for your routes, parameters, and search inputs, making it the top choice for complex TypeScript apps.
