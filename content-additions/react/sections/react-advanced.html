React Hook Form (RHF)

The standard choice for performant, flexible, and extensible forms with easy-to-use validation. Unlike controlled components, RHF uses uncontrolled inputs via refs to prevent expensive UI re-renders on every keystroke.

  • register(): Registers an input element into RHF and applies its validation rules.
  • handleSubmit(): Wraps your submit handler. It automatically passes the validated data or handles preventing submission if there are validation errors.
  • Zod Integration: Strongly recommended to use `@hookform/resolvers/zod` to declare a TypeScript-first schema for validating form shapes.

React Router v7

React Router has evolved from a pure component-based router into a full framework-level routing architecture (merging with Remix concepts).

  • Data Loaders: Move data fetching OUT of your components. Loaders run before the route renders, eliminating "spinner hell" and waterfall fetches.
  • Actions: Data mutations (POST/PUT/DELETE) are handled natively through form submissions routed to specialized Action functions.
  • createBrowserRouter: The modern DOM-based routing setup required to enable loaders, actions, and advanced error boundaries.
Tip: If you are using Next.js App Router, you don't use React Router. But if you are building an SPA (e.g., with Vite), React Router v7 is mandatory for modern data-loading patterns.

Testing & Patterns

Ensuring application reliability and architecting scalable, reusable code.

React Testing Library (RTL)

RTL guides you to test software the way a user interacts with it (finding buttons by text, checking what's visible) rather than testing implementation details (like component state).

  • Roles & Accessibility: `screen.getByRole('button', { name: /submit/i })` enforces accessibility while making tests resilient to DOM changes.
  • user-event: A companion library (`@testing-library/user-event`) that accurately simulates full browser interactions (e.g., firing focus, keydown, keypress, input, keyup events just from typing).
  • Mocking: In modern testing, instead of mocking React components, we mock the Network using Mock Service Worker (MSW). MSW intercepts fetch/axios requests at the network level.
