Web Accessibility (A11y) & Core

Building the web for everyone. Accessibility is not an afterthought; it is fundamental to modern markup.

Semantic HTML Boundaries

Semantic HTML natively provides meaning and keyboard functionality that plain `<div>`s lack.

  • <main>: The dominant content of the page. Only one visible per document.
  • <article> vs <section>: An `article` is independently distributable (a blog post). A `section` groups related content structurally but isn't necessarily standalone.
  • <nav>: Wrap major navigation blocks. Provide an `aria-label` like "Main" or "Footer" if multiple exist.
  • <button> vs <a>: `<a href>` navigates. `<button>` performs an action on the page. Never use `<div onClick>` without adding `tabindex="0"`, `role="button"`, and handling the `Enter/Space` keys manually.
ARIA (Accessible Rich Internet Applications)

ARIA bridges the gap for complex interactive UI (tabs, modals, comboboxes) where native HTML falls short.

  • First Rule of ARIA: No ARIA is better than bad ARIA. Native elements are always preferred.
  • aria-hidden="true": Hides decorative elements (like SVG icons) from screen readers.
  • aria-label / aria-labelledby: Provides an invisible text label for icon-only buttons (like a search magnifying glass).
  • aria-live: Used on dynamically updated areas (e.g. error messages, toast notifications). Screen readers will interrupt and read `"assertive"` updates immediately, or wait for pauses for `"polite"` updates.

Modern CSS Architecture

Recent CSS evolutions drastically reduce the need for preprocessors and complex JS logic.

Game-Changing Selectors
  • :has() (The Parent Selector): Selects an element if it contains a certain descendent. Ex: `figure:has(figcaption)` selects only figures that have captions. It completely changes CSS architecture.
  • :is() and :where(): Groups selectors succinctly. `:is(h1, h2, h3) span` instead of writing them all out. `:where()` is identical but has zero specificity, perfect for library resets.
  • Native Nesting: CSS now natively supports standard nesting (similar to SCSS/LESS) without build tools. `nav { ul { padding: 0; } }`
Layout & Responsiveness
  • Container Queries (@container): Instead of styling based on viewport width (`@media`), style based on the parent wrapper's width. This allows reusable components (like cards) to look totally different inside a tight sidebar vs a massive main grid.
  • CSS Subgrid: Allows nested elements to perfectly align to their parent's grid tracks. If a Card has a Grid layout, `grid-template-rows: subgrid` allows headers and text across multiple sibling cards to perfectly align with each other.
  • text-wrap: balance / pretty: Improves typography natively. `balance` prevents massive headlines with a single "widow" word on the final line by mathematically balancing the text block width.
