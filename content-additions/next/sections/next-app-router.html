App Router Mastery

Deep dive into Next.js's file-system based router built on React Server Components.

Routing Architecture (Parallel & Intercepting)

The App Router supports advanced routing patterns that were previously impossible or extremely difficult in the `pages` directory.

  • Parallel Routes (@folder): Render multiple independent pages in the same layout. Perfect for complex dashboards with separate sections (e.g., Feed, Analytics, Team) that each need their own loading/error states.
  • Intercepting Routes ((.)folder): Hijack navigation to show a route as a modal over the current page, while still allowing the URL to reflect the new route. Useful for photo galleries or detail views.
// Folder Structure
app/
  layout.tsx
  @analytics/  // Parallel Route
    page.tsx
  @team/       // Parallel Route
    page.tsx
  feed/
    page.tsx
Data Fetching & Caching Strategy

Next.js unified data fetching by extending the native `fetch` API with caching and revalidation logic.

  • Segment Config: Use `export const dynamic = "force-dynamic"`, `revalidate = 3600`, etc., to control a route's caching behavior globally.
  • unstable_cache: For caching data that doesn't use `fetch` (like direct database queries or heavy calculations).
  • On-Demand Revalidation: Use `revalidatePath()` or `revalidateTag()` inside a Server Action to purge the cache after a database update.
Tip: Always fetch data in Server Components. It reduces the amount of JavaScript sent to the client and keeps your API keys secret.
