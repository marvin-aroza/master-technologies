Type Gymnastics: Recursion

Using recursive conditional types to create deep, complex object transformations:

// Deep Readonly Pattern
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object 
    ? DeepReadonly<T[P]> 
    : T[P];
};

// JSON Safety Pattern
type JSONValue =
  | string | number | boolean | null
  | { [key: string]: JSONValue }
  | JSONValue[];

Branded Types (Nominal Simulation)

TS is structural, but branding ensures data integrity for IDs, Emails, and sensitive primitives:

type Brand<K, T> = K & { __brand: T };

type UserID = Brand<string, "UserID">;
type OrderID = Brand<string, "OrderID">;

const u: UserID = "123" as UserID;
const o: OrderID = u; // Error: OrderID !== UserID

Mapped Type Remapping

Using the as clause to dynamically rename keys based on templates:

interface User {
  id: number;
  name: string;
}

type Getters<T> = {
  [K in keyof T as `get${Capitalize<string & K>}`]: () => T[K]
};

type UserGetters = Getters<User>;
// { getName: () => string; getId: () => number; }

Satisfies vs Explicit Annotations

The satisfies operator (TS 4.9+) allows validation WITHOUT widening inferred types:

type Palette = Record<string, string | number[]>;

const colors = {
  primary: "#00f",
  secondary: [0, 0, 255]
} satisfies Palette;

// colors.primary remains "#00f" (literal)
// NOT widened to string | number[]
