TypeScript Fundamentals

Extending JavaScript with static typing to build robust, scalable applications.

Types vs Interfaces

Both are used to define custom structures in TypeScript, but they have subtle differences.

  • Interface: Strictly for object shapes. Can be merged via "Declaration Merging" (if you declare the same interface twice, TS combines their properties). Ideal for public API definitions.
  • Type Aliases (`type`): More flexible. Can define primitives, unions, intersections, and mapped types. Cannot be declaration-merged. Ideal for internal component definitions and complex utility types.
// Interface (Object Shape)
interface User { id: number; name: string; }

// Type (Union)
type Status = "idle" | "loading" | "success" | "error";

// Intersection (Combining shapes)
type AdminUser = User & { role: "admin" | "superadmin" };
Generics & Utility Types

Generics allow you to write reusable code that works across multiple types.

  • `T` as an Argument: Think of a generic `` like a type variable passed into a function.
  • Partial<T>: Makes all properties in T optional. Useful for update/patch payloads.
  • Omit<T, K>: Creates a type by removing specific keys `K` from `T`.
  • Record<K, T>: Creates an object type where keys are of type `K` and values are of type `T`.
// Generic Function: infers type T from the input array
function getFirstItem<T>(items: T[]): T | undefined {
  return items[0];
}

const num = getFirstItem([1, 2, 3]); // Hover: num is number
const str = getFirstItem(["a", "b"]); // Hover: str is string
Discriminated Unions (Tagged Unions)

A pattern used to safely exhaust all possible variations of a type by sharing a single common literal property (the "discriminator").

type Action = 
  | { type: "LOADING" } 
  | { type: "SUCCESS"; payload: string[] } 
  | { type: "ERROR"; errorMsg: string };

function reducer(action: Action) {
  // TypeScript knows exactly what properties are available
  // based on the 'type' property.
  switch (action.type) {
    case "LOADING":
      return 'Loading...';
    case "SUCCESS":
      return action.payload.length; // Safe to access payload
    case "ERROR":
      return action.errorMsg; // Safe to access errorMsg
  }
}
