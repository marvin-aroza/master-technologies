Micro-Frontends

Scaling applications across multiple independent teams and codebases.

Module Federation (Webpack) / Native Federation (Vite/Esbuild)

Micro-frontends allow you to split a monolithic Angular app into smaller, independently deployable applications. These applications load each other's compiled code at runtime.

  • Host Application (Shell): The main app that boots up, handles global routing, and loads remote modules.
  • Remote Application: A standalone application that exposes parts of itself (components or routes) to be consumed.
  • Shared Dependencies: Federation allows you to declare dependencies (like `@angular/core` or RxJS) as singletons, so the browser doesn't download the Angular framework 5 times.

Because Angular 17+ moved away from Webpack towards Esbuild/Vite, the standard tool has become @angular-architects/native-federation.

// Host routing loading a remote Micro-Frontend
export const routes: Routes = [
  {
    path: 'flights',
    loadChildren: () =>
      loadRemoteModule('flights', './Module')
        .then(m => m.FlightsRoutes)
  }
];
