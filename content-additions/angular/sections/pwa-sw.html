PWA & Service Workers

Turning Angular apps into installable, offline-capable Progressive Web Apps.

@angular/pwa

Adding `ng add @angular/pwa` instantly configures your Angular app with a Web App Manifest and an optimized Service Worker (`ngsw-worker.js`).

  • ngsw-config.json: The configuration file controlling cache policies.
  • assetGroups: Caches app shell resources (JS, CSS, initial HTML, icons). Can be `installMode: 'prefetch'` (download immediately) or `'lazy'` (download on demand).
  • dataGroups: Caches API requests. Strategies include `performance` (cache-first, network-fallback) and `freshness` (network-first, cache-fallback with timeouts).
  • SwUpdate: Angular service to communicate with the service worker. Checks for updates (`checkForUpdate()`) and tells the SW to activate new versions so the app can prompt the user to refresh.
  • SwPush: Service for subscribing to and receiving Web Push Notifications payload data.
// ngsw-config.json (Data Caching)
"dataGroups": [{
  "name": "api-freshness",
  "urls": ["/api/live-data/**"],
  "cacheConfig": {
    "strategy": "freshness",
    "maxSize": 100,
    "maxAge": "3d",
    "timeout": "5s" // If network takes >5s, serve cached!
  }
}]
