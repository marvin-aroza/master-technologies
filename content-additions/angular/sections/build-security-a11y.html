Build, Security & A11y

Advanced compilation toolchain, security mechanisms, and accessibility standards.

Esbuild & Application Builder (v17+)

Modern Angular uses the `@angular-devkit/build-angular:application` builder, driven by Esbuild and Vite.

  • Esbuild: Replaces Webpack for JS/TS compilation and bundling. Written in Go, it is 2x to 4x faster during builds.
  • Vite: Powers the local dev server (`ng serve`), providing rapid HMR (Hot Module Replacement) and optimized dev-time caching.
  • Bundle Optimization: Angular ensures aggressive Tree-Shaking (removing unused code). Standalone components drastically improve tree-shaking by removing the "heavy" `NgModule` closures.
Security (DomSanitizer & CSRF)

Angular is secure by default, treating all injected values as untrusted.

  • Contextual Escaping: Angular recognizes the context of a binding (HTML, Style, Resource URL) and automatically escapes or cleanses the string to prevent XSS (Cross-Site Scripting).
  • DomSanitizer: If you mathematically know a string is safe (e.g., an iframe URL you control), you can bypass security using `sanitizer.bypassSecurityTrustHtml(value)`. Use with extreme caution.
  • CSRF Protection: `HttpClient` has built-in XSRF (CSRF) protection. If the server sends an `XSRF-TOKEN` cookie, Angular automatically attaches it as an `X-XSRF-TOKEN` HTTP header to all mutation requests (POST/PUT/DELETE).
Accessibility (A11y)

Building for all users requires using ARIA attributes and focus management correctly.

  • ARIA in Templates: Use attribute bindings for ARIA state `[attr.aria-expanded]="isOpen"`. Warning: you must use the `attr.` prefix for ARIA attributes because they are HTML attributes, not DOM properties.
  • CDK LiveAnnouncer: A service to announce messages for screen-reader users (perfect for dynamic updates like "Item saved" or "Notification received").
  • CDK FocusTrap: Crucial for modals. Prevents keyboard navigation (Tab key) from leaving the modal while it is open.
