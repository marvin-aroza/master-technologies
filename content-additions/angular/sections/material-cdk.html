Building accessible, high-quality UI components leveraging the Component Dev Kit.

Component Dev Kit (CDK)

The CDK provides behavior primitives without any visual styling. It's the foundation of Angular Material but can be used to build your own custom design systems.

  • Overlay: Used for rendering floating panels safely (dialogs, tooltips, menus).
  • Portal: Dynamically render templates or components anywhere in the DOM.
  • A11y: Accessibility utilities like `FocusTrap`, `LiveAnnouncer`, and `ListKeyManager`.
  • Drag and Drop: Declarative drag/drop interfaces with custom preview and drop zones.
  • Scrolling: Virtual scrolling with `cdk-virtual-scroll-viewport` to render large lists efficiently.
  • Layout: `BreakpointObserver` to evaluate media queries with RxJS.
// Example: Virtual Scrolling for 100k items
import { ScrollingModule } from '@angular/cdk/scrolling';

@Component({
  standalone: true,
  imports: [ScrollingModule],
  template: `
    
      

        {{item}}
      

    
  `,
  styles: ['.viewport { height: 400px; width: 100%; }']
})
export class VirtualList {
  items = Array.from({length: 100000}).map((_, i) => `Item #${i}`);
}
Angular Material v17+ (M3)

Angular Material implements Google's Material Design 3 (M3) specification. It provides fully accessible and themeable components.

  • Design Tokens: M3 relies heavily on CSS variables (custom properties), allowing for dynamic theming at runtime without rebuilding Sass.
  • Typography & Density: Fully customizable scales that automatically push down into all components.
  • Key Components: `MatTable` (complex data grids), `MatDialog` (modals via Overlay), `MatFormField` (advanced input wrapper with error states).
Tip: Never use `::ng-deep` to style Material components. Always use the provided CSS variables or design tokens to override styles safely.
